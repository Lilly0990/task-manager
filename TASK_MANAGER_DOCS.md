## ЩО ЦЕ ТАКЕ

Task Manager — це веб-застосунок для роботи з клієнтськими задачами. Суть проста:

- **Адмін** (ти) створює клієнта і отримує унікальне посилання
- **Клієнт** відкриває це посилання і пише задачі — текст, скріншоти, файли
- **Адмін** бачить всі задачі, може їх редагувати, позначати виконаними або видаляти
- При кожній новій задачі приходить **повідомлення в Telegram**

Реєстрації для клієнта немає — тільки посилання. Посилання = доступ.

## ДВІ РОЛІ В СИСТЕМІ

### Адмін
- Заходить через `/admin` з паролем
- Бачить список всіх клієнтів
- Створює нових клієнтів
- Переглядає і керує задачами кожного клієнта

### Клієнт
- Відкриває своє унікальне посилання — без пароля, без реєстрації
- Бачить тільки свої задачі
- Додає нові задачі, редагує і видаляє свої

## КРОК ЗА КРОКОМ — СЦЕНАРІЙ АДМІНА

### 1. Вхід в адмін-панель

```
Відкрити: /admin
Ввести пароль → натиснути "Увійти"
```

Пароль зберігається в Supabase (таблиця `settings`, ключ `admin_password`).
Після входу — cookie сесія, залишається залогіненим у браузері.

---

### 2. Головна сторінка адміна — список клієнтів

Після входу відкривається список всіх клієнтів. Для кожного видно:
- Ім'я клієнта
- Лічильник задач: скільки виконано / скільки всього (наприклад `3/7`)

Кнопки у хедері:
- **Пароль** — змінити пароль адміна
- **Вийти** — завершити сесію

---

### 3. Створити нового клієнта

```
1. Ввести ім'я або назву компанії у поле "Новий клієнт"
2. Натиснути "Створити"
3. З'явиться зелений блок з унікальним посиланням
4. Скопіювати посилання кнопкою "Копіювати"
5. Надіслати клієнту
```

Посилання має вигляд:
```
https://task-manager-two-lilac-71.vercel.app/xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
```

Токен — це UUID, генерується автоматично при створенні клієнта. Один клієнт = одне посилання назавжди.

---

### 4. Переглянути задачі клієнта

```
1. Натиснути на клієнта у списку
2. Відкривається сторінка /admin/[clientId] з усіма задачами
3. Вгорі — кнопка "Копіювати посилання клієнта" (зручно надіслати повторно)
4. Задачі згруповані по датах: Сьогодні / Вчора / 15 березня і т.д.
```

---

### 5. Що адмін може робити з задачами

**Відмітити виконаною:**
Натиснути на квадратик зліва від задачі → стає зеленим, задача сіріє і перекреслюється

**Редагувати:**
Клікнути на текст задачі → відкривається редактор → змінити → "Зберегти"

**Видалити:**
Натиснути × справа від задачі → задача видаляється разом з усіма прикріпленими файлами зі сховища

## КРОК ЗА КРОКОМ — СЦЕНАРІЙ КЛІЄНТА

### 1. Відкрити своє посилання

Клієнт отримує посилання від адміна і просто відкриває його у браузері. Без реєстрації, без пароля. Відкривається сторінка з його ім'ям і списком задач.

---

### 2. Додати нову задачу

```
1. У верхньому полі написати текст задачі
2. Можна вставити скріншот: Ctrl+V прямо в текстове поле
   (зображення завантажується в хмару і вставляється inline)
3. Можна прикріпити файли: натиснути "Файли" → вибрати файли
   Підтримувані формати: PDF, DOC, DOCX, ZIP, TTF, OTF, WOFF, WOFF2, TXT
   Ліміти: до 5 файлів, кожен до 50MB
4. Натиснути "Зберегти"
```

Після збереження:
- Задача з'являється у списку згори
- Адмін отримує повідомлення в Telegram

---

### 3. Telegram-повідомлення (що приходить адміну)

При кожній новій задачі у Telegram приходить:
```
📋 Нова задача
[Ім'я клієнта]

🕐 23.03.2026, 14:35

🔗 https://task-manager.../[токен клієнта]
```

---

### 4. Що клієнт може робити з задачами

**Відмітити виконаною:** натиснути квадратик зліва

**Редагувати:** клікнути на текст → редактор → "Зберегти"

**Видалити:** натиснути × справа

## ЯК ВЛАШТОВАНИЙ РЕДАКТОР ЗАДАЧ

Редактор підтримує два типи блоків:

| Блок | Як додати |
|------|-----------|
| Текст | Просто пишеш |
| Зображення | Ctrl+V або перетягнути |

Зображення завантажуються моментально в Supabase Storage і вставляються прямо між текстом. Задача зберігається як JSON-масив блоків:

```json
[
  { "type": "text", "value": "Потрібно виправити кнопку на головній" },
  { "type": "image", "url": "https://...supabase.co/storage/..." },
  { "type": "text", "value": "Ось скріншот як виглядає зараз" }
]
```

Окремо від inline-зображень є кнопка **Файли** — для вкладень (PDF, документи тощо). Вони зберігаються окремо і відображаються як посилання під задачею.

## ЯК ВЛАШТОВАНА БАЗА ДАНИХ (Supabase)

**Таблиця `clients`**

| Поле | Що зберігає |
|------|-------------|
| `id` | Унікальний ID клієнта |
| `name` | Ім'я або назва компанії |
| `token` | UUID — це і є унікальне посилання |
| `created_at` | Дата створення |

**Таблиця `tasks`**

| Поле | Що зберігає |
|------|-------------|
| `id` | Унікальний ID задачі |
| `client_id` | Прив'язка до клієнта |
| `content` | JSON з блоками (текст + зображення) |
| `files` | Масив URL вкладених файлів |
| `is_done` | true / false |
| `date` | Дата задачі (YYYY-MM-DD) |
| `created_at` | Точний час створення |

**Таблиця `settings`**

| Ключ | Що зберігає |
|------|-------------|
| `admin_password` | Пароль для входу в /admin |

**Storage bucket `task-files`**
Публічне сховище для всіх файлів і inline-зображень. Ліміт файлу: 50MB.

## API — ЩО ВІДБУВАЄТЬСЯ ПІД КАПОТОМ

| Запит | Що робить |
|-------|-----------|
| `GET /api/clients` | Повертає список всіх клієнтів з кількістю задач |
| `POST /api/clients` | Створює нового клієнта, генерує UUID токен |
| `POST /api/tasks` | Зберігає задачу, завантажує файли, надсилає Telegram |
| `PATCH /api/tasks/[id]` | Оновлює задачу (текст або статус виконання) |
| `DELETE /api/tasks/[id]` | Видаляє задачу + всі файли з Storage |
| `POST /api/admin/login` | Перевіряє пароль, встановлює cookie |
| `POST /api/admin/logout` | Видаляє cookie |
| `POST /api/admin/password` | Змінює пароль в Supabase settings |
| `POST /api/upload` | Завантажує inline-зображення (Ctrl+V) |

## НАЛАШТУВАННЯ (ENV ЗМІННІ)

Зберігаються у Vercel Dashboard та локально в `.env.local`:

| Змінна | Що це |
|--------|-------|
| `NEXT_PUBLIC_SUPABASE_URL` | URL Supabase проєкту |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Публічний ключ Supabase |
| `SUPABASE_SERVICE_ROLE_KEY` | Серверний ключ (для адмін-операцій) |
| `TELEGRAM_BOT_TOKEN` | Токен Telegram бота |
| `TELEGRAM_CHAT_ID` | ID чату куди надсилаються повідомлення |

## ТИПОВІ СИТУАЦІЇ

**Клієнт загубив посилання**
Зайти в /admin → натиснути на клієнта → "Копіювати посилання клієнта" → надіслати знову

**Потрібно змінити пароль адміна**
Зайти в /admin → кнопка "Пароль" у правому верхньому куті → ввести новий → "Зберегти"

**Telegram повідомлення не приходять**
Перевірити env змінні `TELEGRAM_BOT_TOKEN` і `TELEGRAM_CHAT_ID` у Vercel Dashboard

**Файл не завантажується**
Перевірити: розмір до 50MB, формат підтримується (PDF, DOC, DOCX, ZIP, TTF, OTF, WOFF, WOFF2, TXT), не більше 5 файлів за раз
